# Changelog

## v1.0.0 - Portfolio Release

### Completed Phases
- Phase 0: repository setup
- Phase 1: data loading, cleaning, and EDA
- Phase 2: feature engineering and baseline modeling
- Phase 2.5: model hardening, threshold reporting, reproducible artifacts
- Phase 3: FastAPI prediction service
- Phase 4: Streamlit risk monitoring dashboard
- Phase 5: rule-based alert engine
- Phase 6: local production readiness
- Phase 7: portfolio documentation and interview guide
- Phase 8: demo assets and publication materials
- Phase 9: technical cleanup and repository hardening
- Phase 10: visual polish and v1.0 readiness

### Major Capabilities
- End-to-end fraud risk workflow from data prep to serving and monitoring
- Reproducible sklearn modeling and threshold trade-off reporting
- FastAPI endpoints for prediction, metadata, and alert operations
- Streamlit dashboard for operational visibility
- Rule-based alerting with local JSONL persistence
- Docker Compose + Makefile + CI for local reproducibility

### Known Limitations
- Local deployment only
- No authentication/authorization
- No managed production database
- No real-time streaming infrastructure
